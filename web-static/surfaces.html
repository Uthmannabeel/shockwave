Where it runs

One graph. Five surfaces.

Impact analysis is only useful where review actually happens. Shockwave ships as a command, a bot, an agent, a map, and a reusable skill — all reading the same Orbit graph.

⌨️

Command line

shockwave analyze <symbol|file> and shockwave diff <ref> — against Orbit Local or, with --remote, the hosted graph. Markdown, JSON, an interactive HTML report, or a Mermaid graph.

🗺️

Blast Monitor

The interactive radial map: the change is the epicenter, impact ripples outward in depth-colored rings, and public entry points glow cyan with their call path traced to the core.

Merge-request bot

A GitLab CI job that, on every merge request, computes the blast radius from Orbit Remote and posts (and updates) a review comment — summary, untested hotspots, and the suggested tests.

🧩

Orbit Reachability skill — for the ecosystem

Orbit's traversal DSL is powerful but unforgiving: every query must be anchored, full scans are rejected, and cross-file calls are pre-resolved. We packaged the patterns we learned — the rules, ready JSON recipes, and the hop-by-hop reachability algorithm — as a reusable skill, so any Duo agent or MCP client can do correct graph reachability over Orbit without re-learning the gotchas.

Why a knowledge graph

grep finds text. Orbit knows your code.

Text search

  • Matches a name, not a reference
  • Misses cross-file and inherited callers
  • No notion of distance, risk, or coverage
  • Can't tell internal from externally reachable

Shockwave + Orbit

  • Follows the resolved call & inheritance graph
  • Transitive, cycle-safe, with hop distance
  • Ranks risk and flags untested hotspots
  • Traces the path from a public entry point to the change
