Get started in two minutes.

Index a repo with Orbit, install Shockwave, and run your first blast-radius analysis.

1 · Index a repo with Orbit Local

Install the Orbit CLI, then index any repository into a local graph.

shell
# macOS / Linux
curl -fsSL https://gitlab.com/gitlab-org/orbit/\
knowledge-graph/-/raw/main/install.sh | bash

# Windows (PowerShell)
irm https://gitlab.com/gitlab-org/orbit/\
knowledge-graph/-/raw/main/install.ps1 | iex

orbit index .

2 · Install Shockwave

Python 3.11+. The web UI is an optional extra.

shell
pip install -e .

# with the Blast Monitor web UI
pip install -e ".[web]"

3 · Analyze

shell
# impact of a symbol or file
shockwave analyze setupmethod --format md

# blast radius of a whole diff (great in CI)
shockwave diff main

# against the hosted graph — Orbit Remote
shockwave analyze compute --remote https://gitlab.com --token $GITLAB_TOKEN

# the interactive map
shockwave-web        # → http://127.0.0.1:7777

Honesty

What Shockwave is — and isn't.

A tool you can trust is one that's clear about its edges. Shockwave is exact about what the graph says and explicit about what it can't see.

Inherent to static analysis

It follows Orbit's resolved CALLS/EXTENDS edges, so it can't see dynamic dispatch, reflection, callbacks, or config-wired calls. Value/constant changes (no call edge) and config/non-code changes fall outside the code graph.

Coverage is "no direct test", not real coverage; exposure and risk are signals, not guarantees.

Bounded by Orbit

Results reflect Orbit's index: the default branch, supported languages, and — on Remote — only code your token can see (permission-aware, so a radius can be partial). There's no cross-repo call graph yet.

The depth is capped per run, and very large fan-outs may be truncated — Shockwave tells you when a result is partial or a name was ambiguous, rather than hiding it.

Security reachability (ranking vulnerabilities by whether they're reachable) is the headline item on the roadmap — Orbit's Finding nodes don't yet carry a code location, so it needs a join with GitLab's Vulnerability API.
